# Security

Security posture for **netbox-orb-extreme-platformone**: the Extreme Platform ONE
→ NetBox Orb worker, plus the optional local NetBox/Diode compose stack under
`.devcontainer/`.

Report vulnerabilities privately to the repository maintainers. Do not open a
public issue with live credentials, inventory exports, or exploit details.

---

## Scope

| Surface | Role |
|---------|------|
| `orb_extreme_platformone/` | Production worker: Platform ONE HTTP client, NetBox bootstrap REST, Diode entity mapping |
| `agent.yaml` / `workers.txt` | Orb Agent policy + install path (env-substituted secrets) |
| `.devcontainer/` | **Local demo only** — NetBox + Diode + workspace Dev Container; not a production deployment template |
| `.github/workflows/ci.yml` | Lint + pytest on PRs |

---

## Trust model

1. **Operator-controlled endpoints.** `PLATFORMONE_API_URL` and `NETBOX_API_URL`
   are intentional “call this API” knobs. Treat them like credentials: a
   malicious URL is SSRF by design for an API client.
2. **Secrets live in the environment**, not in git. Policy YAML may reference
   `${VAR}` placeholders; Orb substitutes at runtime.
3. **Bootstrap is one-shot and privileged.** `BOOTSTRAP: true` uses a NetBox
   token that can create/update custom fields and tags. Scheduled runs should
   keep `BOOTSTRAP: false` and drop `NETBOX_API_*`.
4. **The local stack is insecure by design for convenience** (`admin`/`admin`,
   CORS wide open, Hydra `--dev`, TLS verify skipped between Diode and local
   HTTP NetBox). It must stay on loopback and off shared/prod networks.

## What we harden (worker)

### HTTPS and URL hygiene

`orb_extreme_platformone.urls.require_https_url` gates every outbound base URL:

- Remote hosts require `https://`.
- Plaintext `http://` is allowed only for loopback, `*.local`, and the explicit
  compose hostname `netbox` (so Dev Container bootstrap can use
  `http://netbox:8080`).
- Userinfo is rejected (`https://legit@evil.com`, `user:pass@host`) to block
  URL-confusion credential theft.
- Query strings and fragments are rejected.

### HTTP client

- TLS certificate verification left at requests defaults (`verify=True`).
- Timeouts on all Platform ONE and NetBox calls.
- Redirects disabled (`allow_redirects=False`) so a `307` cannot replay a
  login password body or carry a NetBox token off-origin.
- Platform ONE error bodies are truncated (`truncate_error_body`, 200 chars)
  before they enter exceptions/logs.

### Bootstrap fail-closed

When policy `BOOTSTRAP` is true but `NETBOX_API_URL` / `NETBOX_API_TOKEN` are
missing, the worker **raises** instead of silently skipping schema setup.

### No classic injection sinks

The package does not use `subprocess`, `shell=True`, `eval`/`exec`, `pickle`,
or unsafe YAML load. ConfigState table names come from fixed catalogs, not
from free-form policy strings.

### CI

`.github/workflows/ci.yml` sets `permissions: contents: read`, uses
`pull_request` (not `pull_request_target`), and runs only lint/tests.

## Credential handling

| Secret | Where it should live | Notes |
|--------|----------------------|-------|
| `PLATFORMONE_API_TOKEN` or username/password | Agent env / `.env` (gitignored) | Prefer API token over long-lived password when available |
| `DIODE_CLIENT_ID` / `DIODE_CLIENT_SECRET` | Agent env | Diode OAuth client-credentials |
| `NETBOX_API_TOKEN` | Env **only while** `BOOTSTRAP: true` | Least privilege: custom fields + tags; not a permanent superuser token |
| Local stack secrets | `.devcontainer/.env.local`, `.devcontainer/diode/.env`, `.devcontainer/netbox/env/*.env`, OAuth JSON | Generated by `.devcontainer/setup.sh`, mode `600`, gitignored |

**Do not** commit dry-run JSON, inventory exports, or `.env` files. Dry-run
output can contain hostnames, serials, MACs, and management IPs.

Policy config currently wins over environment when both are set
(`_cfg_or_env`). Prefer leaving secret values out of checked-in YAML and
supplying them only via the environment so Orb policy objects never carry
plaintext secrets.

## Local NetBox / Diode stack (`.devcontainer/`)

### Intended use

Exercise the worker against a real Diode ingest path on a developer machine
(Docker/Podman). See `.devcontainer/README.md`.

### Controls in place

- Published ports bind to **127.0.0.1** only (`8000` NetBox, `8080` Diode).
- DB/Redis are not published to the host.
- Secrets are generated with `/dev/urandom`, written mode `600`, and
  **preserved** on re-run of `setup.sh` (no silent password rotation against
  live volumes).
- `agent.local.yaml` is created once and not clobbered (avoids flipping
  `BOOTSTRAP` back to `true`).
- Diode nginx authenticates ingest/reconciler gRPC with `auth_request`.
- NetBox Diode plugin secret is mounted read-only under `/run/secrets`.
- Dev Container runs `./.devcontainer/setup.sh` as host `initializeCommand`
  before compose starts; workspace image tags stay unpinned (rebuild when the
  Dockerfile changes).
- NetBox MCP launcher (`.devcontainer/netbox-mcp.sh`) embeds no secrets; it
  reads gitignored `.env.local` and pins the MCP package to a release tag.

### Known weak defaults (local only)

| Item | Why it exists | Operator duty |
|------|---------------|---------------|
| NetBox `admin` / `admin` | Demo UX | Never expose beyond loopback; change if the host is shared |
| `CORS_ORIGIN_ALLOW_ALL=True` | Local UI/API convenience | Do not copy into production NetBox |
| `NETBOX_DIODE_PLUGIN_SKIP_TLS_VERIFY=true` | Local HTTP NetBox | Keep `false` against real HTTPS NetBox |
| Hydra `serve all --dev`, Postgres `sslmode=disable` | Upstream Diode compose | Not a production IdP template |
| Floating `latest` Diode/nginx/redis tags + `pull_policy: always` | Track upstream easily | Pin digests/tags for a locked lab |

### MCP privilege note

Docs describe NetBox MCP as read-only, but `create-netbox-token.sh` mints an
**admin** token into `.env.local`. Prefer a dedicated read-only NetBox token
for MCP and keep bootstrap/write tokens separate.

## Historical notes

- Inventory data once landed in git via a committed-then-deleted `test.json`
  (purged from branch history; older PR refs may still hold copies — treat as
  sensitive and request GitHub removal if needed).
- Early `.devcontainer/netbox/env/*.env` commits used shared demo credentials;
  those files are gitignored at tip. Do not reuse values from history for
  anything beyond a disposable local volume.

## Residual risks / backlog

Ordered by practical impact:

1. **Pin** Diode / nginx / redis / `orb-agent` images (digest or immutable tag)
   for reproducible labs; drop blanket `pull_policy: always` when pinned.
2. **Randomize** the local NetBox superuser password (print once) instead of
   hard-coded `admin`.
3. **Mint a read-only** NetBox token for MCP separate from bootstrap/admin.
4. Add a **lockfile** (or exact pins) for production worker installs; current
   `pyproject.toml` uses lower bounds only.
5. Prefer **env-only secrets** in Orb policy (stop winning from `config:` for
   credential keys) if Orb can guarantee env substitution without YAML values.
6. Align Codespaces/Dev Container **port visibility** to private/localhost
   where the host product supports it (`forwardPorts` still present for IDE UX).

## Operator checklist

- [ ] Secrets only in env / secret store — never in git or screenshots
- [ ] `BOOTSTRAP: false` on all scheduled runs; NetBox bootstrap token removed
- [ ] `PLATFORMONE_API_URL` / `NETBOX_API_URL` point at expected HTTPS hosts
- [ ] Dry-run / inventory output kept out of the repo
- [ ] Local compose ports remain on `127.0.0.1`; stack not published to a LAN
- [ ] After first local bootstrap, set `BOOTSTRAP: false` in `.devcontainer/agent.local.yaml`
